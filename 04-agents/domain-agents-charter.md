# Future Domain Agents — Charters

> profile을 지금 만들지 않는다. 각 도메인의 **charter(임무·트리거·승격 기준)**만 정의하고, recurring 워크플로가 생기면 그때 Hermes profile화. (DFXISP 루프 1회 증명 이후 확장 — 정본 로드맵.)

## 공통 승격 기준
charter → profile 전환 조건: ① 반복되는 작업 흐름 존재 ② 전용 도구/데이터 필요 ③ 사람이 정기적으로 위임. 셋 중 2+.

## Charters
- **Life Agent** — 생활·루틴·건강·일정. 트리거: "이번 주 일정/루틴 정리". 데이터: 캘린더.
- **Finance Agent** — 예산·지출·투자·수익화. 트리거: "지출 점검/예산". 데이터: 가계/계좌(읽기). 위험작업 승인 필수.
- **Business Agent** — 사업 기획·시장조사·제품화. 트리거: "아이디어 검증/시장 조사".
- **Career Agent** — 커리어·포트폴리오·이력. 트리거: "포트폴리오/이력 업데이트". DFXISP·Agent OS를 자산화.
- **Learning Agent** — 학습계획·복습·skill tree. 트리거: "학습 로드맵/복습".

## 현재 액션
없음(charter 보존). 첫 후보 = **Career Agent**(DFXISP/Agent OS 결과물을 포트폴리오로) — production candidate로 등록 가능. 나머지는 트리거 발생 시.
